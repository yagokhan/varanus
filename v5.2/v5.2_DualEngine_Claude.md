### 2.2 Long Runner (Optimized — v5.2)

All long trades (direction == 1) use a separate, independently-optimized parameter set:

| Parameter | Search Range | Rationale |
|---|---|---|
| `conf_thresh_long` | [0.680, 0.850] | Lower gate → higher frequency, catches reversals |
| `tp_mult_long` | [2.5, 4.5] | Smaller, "step-like" targets for trending longs |
| `sl_mult_long` | [0.60, 1.00] | Protective stop tuned for reversal structures |

**Why asymmetric?** Short setups in crypto exploit liquidity sweeps above highs
(sharp, high-conviction breakdowns). Long setups after selloffs are more gradual,
mean-reverting — they need a lower confidence gate, tighter TP target, and a
well-placed stop below the sweep low.

## Phase 3 — HTF Bias Override ("The Oversold Exception") + Volume Injection

### Problem
The 1D MSS (HTF bias) filter blocks long entries when the macro trend is bearish.
In deep selloffs, this creates a **filtering deadlock** — the 4h structure forms
bullish MSS signals but the 1D bias == -1 suppresses them. Previous thresholds
were too restrictive, catching only extreme capitulation events and generating
only ~17 long trades across 5 folds.

### Solution: `bias_bypass_long` Feature (17th) + `ssl_sweep_long` Feature (18th)

**File:** `varanus/pa_features.py`
**SHORT HUNTER LOCK-DOWN:** All changes here are strictly long-side only.

#### `bias_bypass_long` — Aggressive Bias Bypass (v5.2 Volume Injection)

**Config (updated):**
```python
BIAS_BYPASS_CONFIG = {
    "rsi_1d_threshold":       45,    # v5.2 VI: relaxed from 35 → 45
                                     # Catches pullbacks in neutral-to-bullish
                                     # regimes, not just extreme crashes.
    "price_range_bottom_pct": 0.25,  # v5.2 VI: relaxed from 10% → 25%
                                     # Allows dip-buying earlier in corrections.
    "range_lookback_4h":      600,   # 100 days × 6 4h bars (unchanged)
    "ssl_sweep_mult_long":    0.80,  # SSL sweep threshold multiplier (new)
}
```

**Bypass Condition:**
```
bypass_active = (RSI_1D < 45) OR (close in bottom 25% of its 100-day H-L range)
```

**What changed:**
- `RSI_1D < 45` (was 35): captures pullbacks in choppy/sideways regimes, not just crashes
- Bottom 25% of range (was 10%): dip-buying triggers earlier in the correction cycle
- These thresholds interact with the XGBoost model — it still learns *which* bypass
  bars to trade based on all 18 features combined

#### `ssl_sweep_long` — Sell-Side Liquidity Sweep (18th feature, new)

**What it does:**
Detects downward sweeps of prior swing lows with a **20% relaxed** `min_sweep_pct`
threshold. A bar qualifies if: `low < prior_low × (1 - sweep_min_pct × 0.80)` AND
`close > prior_low` (reversal close).

**Why:**
The standard FVG sweep threshold may miss genuine SSL setups where the wick
is slightly shallower. This feature expands the bypass candidate pool to include
"partial sweep" entries for dip buying, without weakening the full FVG requirement
for the standard (non-bypass) long signal path.

**Effect on `build_dual_labels()`:**
```python
# Before: bypass only if bullish FVG present
bypass_candidates = (bias_bypass_long==1) & (fvg_type==1) & (mss==0)

# After: bypass if bullish FVG OR relaxed SSL sweep
bypass_candidates = (bias_bypass_long==1) & ((fvg_type==1) | (ssl_sweep_long==1)) & (mss==0)
```

**Goal:** Increase long training label count from ~17 → 50–100 across 5 folds,
giving the XGBoost long model sufficient data to learn valid dip-buying patterns.

## Phase 4 — Long Runner Optuna Objective

### Objective Function: Long Runner Score

```
Long Runner Score (per fold) = long_win_rate_pct × max(long_sharpe, 0)

- Maximises both win frequency AND consistency of returns
- Hard gate: long_win_rate < 38% → fold returns -10.0 (penalty, not -999 to preserve pruning)
- DD penalty: if fold MaxDD > 12% → score × 0.40 (same as Hunter)
- Floor: if total long trades < 20 across all folds → return -999.0
```

**Why Win Rate × Sharpe?**
- Win rate alone can be gamed by tiny wins / huge losses
- Sharpe alone can be driven by a few large-pnl outliers
- Their product rewards strategies that win consistently AND generate stable returns

### Search Space (3 Parameters — v5.2 Volume Injection)

```python

                                         # Explores looser stops for reversal trades
```

**Signal Density Target:** 50–100 long trades across the 5-fold backtest period.
